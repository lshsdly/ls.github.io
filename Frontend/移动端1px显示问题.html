      # 移动端 1px 显示问题

      # 起由

      开发公司给加盟商用的 SaaS 系统移动端页面时发现，页面设置的部分 1px 宽度的线条显示并不尽如人意，在多数手机上显示为 2 个像素。
      想到这种问题绝非孤例，本着不再造轮子的想法，在网上借鉴了一些前辈的分析，然后对此问题进行了处理。

      • 适配布局方式：rem + em 布局
      • 问题发现时间：约在开始开发两周后

      下面进入问题正题，由几个像素概念探究这种情况出现的原因。

      # 移动端主要像素概念

      # 1. device pixels 设备像素/物理像素

      • 显示屏幕的最小物理单位
      • 每个 dp 包含自己的宽高、颜色、亮度等属性可供设置
      • 不可再细分

      设备像素是在设备出厂是设定的，设备一旦造出来就不会变大小和数量。例如iPhone5的分辨率是640x1136px,代表屏幕由640行,1136列像素点组成。

      # 2. CSS pixels 逻辑像素/CSS 像素

      CSS 和 JS 所理解的像素单位，为 Web 开发者创造的，在 CSS 和 JavaScript 中使用的一个抽象的层。

      • CSS 声明和几乎所有的 Javascript 属性都使用 CSS 像素，我们通过 CSS 和 javascript 代码设置的像素都是逻辑像素
      • 用户的缩放比会影响单位逻辑像素点对应的实际物理像素的多少，只有 zoom 100% 的情况下，1 个逻辑像素才会等于 1 个设备像素
      • 逻辑像素是可以被硬件和软件任意调节的单位

      PS. 在没有 retina 屏之前，设备独立像素与物理像素是相等的。

      我们平时使用 Chrome 的设备调试工具的时候，iPhone6 是高 667px，宽是 375px，与苹果官方的 1334px x 750px，长宽分别少了 2 倍，那么面积就少了 4 倍。这就是经常说的 Retina 屏幕用四个(物理)像素表示一个(逻辑)像素。

      # 3. PPI 像素密度

      Pixels per inch，每英寸像素，代指是每英寸中有多少个物理像素。

      • 像素密度越高，代表屏幕显示效果越精细；
      • 而为了避免因为分辨率成倍提高造成高分辨率屏显示内容过小的问题，retina 屏将 2x2 的像素当做 1 个像素来使用。

      # 4. DPI 设备像素比

      Device Pixel Ratio，设备像素比

      • 设备像素比dpr = 设备像素 / CSS像素(某一方向上)

      可以通过 window.devicePixelRatio 获取设备的 dpr 值。

      一般来说，在桌面的浏览器中，设备像素比（dpr）等于 1，一个 css 像素就是代表的一个物理像素。
      而在移动端，大多数机型都不是为1，如 iphone 6、7、8 的 dpr 为 2，那么一个设备独立像素便为 4 个物理像素，因此在 css 上设置的 1px 在其屏幕上占据的是 2 个物理像素，0.5px 对应的才是其所能展示的最小单位。

      由是得知，逻辑像素与设备硬件像素不完全对应，1 逻辑像素对应 2 or 3 设备像素，导致 1px 的显示效果与预期有异。

      早先的移动设备中，并没有 DPR 的概念。随着技术的发展，移动设备的屏幕像素密度越来越高。从 iPhone4 开始，苹果公司推出了 Retina 视网膜屏幕。之所以叫做视网膜屏幕，是因为屏幕的 PPI 太高，人的视网膜无法分辨出屏幕上的像素点。iPhone4 的分辨率提高了一倍，但屏幕尺寸却没有变化，这意味着同样大小的屏幕上，像素多了一倍，于是 dpr = 2。

  document.body.removeChild(testElem);

      # 2. 图片替换

      # border-image

      这种是 AlloyTeam 的《CSS 实现类似原生效果的 1px 边框》这篇文章上看到的，代码写起来挺简单，但是要自己制作图片，而且圆角也不好弄，如果改了颜色就要对图片处理，所以不是很好的方案。

      准备符合需要的图片，做对应的设置，如底边 1px 的图片需要为：

      • 高度 2px
      • 上部透明，下部 1px 需要颜色的填充
      • 对应代码如下：
      .bd1px-btm {
  border-width: 0 0 1px 0;  /* 下边框 */
  -webkit-border-image: url(linenew.png) 0 0 2 0 stretch;
  border-image: url(linenew.png) 0 0 2 0 stretch;
}

      # 3. 多背景渐变

      与 background-image 类似，将图片替换为css3渐变。

      • 设置1px的渐变背景，50%有颜色，50%透明 -> CSS 替代图片

  background:
    linear-gradient(180deg, black, black 50%, transparent 50%) top    left  / 100% 1px no-repeat,
    linear-gradient(90deg,  black, black 50%, transparent 50%) top    right / 1px 100% no-repeat,
    linear-gradient(0,      black, black 50%, transparent 50%) bottom right / 100% 1px no-repeat,
    linear-gradient(-90deg, black, black 50%, transparent 50%) bottom left  / 1px 100% no-repeat;

      # 5. viewport + rem

      这是淘宝的flexible提出的解决方案，其核心就是根据 window.devicePixelRatio（dpr）的值动态改变 viewport 的缩放，核心代码如下（有删减）：

      if (!dpr && !scale) {
  var isAndroid = win.navigator.appVersion.match(/android/gi);
  var isIPhone = win.navigator.appVersion.match(/iphone/gi);
  var devicePixelRatio = win.devicePixelRatio;
  if (isIPhone) {
    // iOS下，对于2和3的屏，用2倍的方案，其余的用1倍方案
    if (devicePixelRatio >= 3 && (!dpr || dpr >= 3)) {
      dpr = 3;
    } else if (devicePixelRatio >= 2 && (!dpr || dpr >= 2)){
      dpr = 2;
    } else {
      dpr = 1;
    }
  } else {
    // 其他设备下，仍旧使用1倍的方案
    dpr = 1;
  }
  scale = 1 / dpr;
}
if (!metaEl) {
  metaEl = doc.createElement('meta');
  metaEl.setAttribute('name', 'viewport');
  metaEl.setAttribute('content', 'initial-scale=' + scale + ', maximum-scale=' + scale + ', minimum-scale=' + scale + ', user-scalable=no');
  if (docEl.firstElementChild) {
    docEl.firstElementChild.appendChild(metaEl);
  } else {
    var wrap = doc.createElement('div');
    wrap.appendChild(metaEl);
    doc.write(wrap.innerHTML);
  }
}

      viewport 缩放原理： viewport 默认宽度是 980px；设置了 initial-scale 则宽度是 device-width/initial-scale；设置了 width 则宽度等于 width 的值；同时设置了 initial-scale 和 width 则宽度取两者中较大的一个。

      这种兼容方案相对比较完美，适合新的项目，老的项目修改成本过大。

      # 6. 伪类 + transform

      把原先元素的 border 去掉，然后利用 :before 或者 :after 重做 border ，并 transform 的 scale 缩小一半，原先的元素相对定位，新做的 border 做绝对定位。

      • 单条 border：
      .scale-1px{
  position: relative;
  border:none;
}
.scale-1px:after{
  content: '';
  position: absolute;
  bottom: 0;
  background: #000;
  width: 100%;
  height: 1px;
  -webkit-transform: scaleY(0.5);
  transform: scaleY(0.5);
  -webkit-transform-origin: 0 0;
  transform-origin: 0 0;
}

      # 解决后遇到的小问题

      在部分列表页面，不能显示边框。

      • KO 掉小问题
        • 比较尴尬，是因为 :after 元素在页面 z 轴的层级跟随本元素，导致更高层级的元素遮盖，提升伪元素的 z-index 后就能正常显示了。
